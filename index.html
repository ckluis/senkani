The idea

Everything works. Out of the box.

Every developer tool ships with the same implicit contract: you will spend a weekend configuring this before it does anything for you. Dotfiles, plugin managers, YAML, TOML, JSON, shell profiles, environment bootstrappers — a Jenga tower of config that has to be rebuilt on every new machine.

Senkani rejects the contract. Install the binary. Open it. A terminal is running. Your project is indexed. 13 MCP tools are registered with Claude Code. The markdown editor renders. The analytics pane is tracking your token spend. You did not configure any of this.

The philosophy is short: if the tool needs config to be useful, the tool is broken.

The intelligence layer

50–90% fewer tokens.
No workflow changes.

Every tool call your AI makes passes through Senkani first. File reads, shell commands, symbol lookups — each is intercepted, compressed, and returned with only what the AI actually needs. The AI doesn't know anything changed. You save 50–90% of your context budget on every session.

The pane system

Working panes,
infinite layouts.

A senkani project is a horizontally-scrollable canvas of panes. Each pane is one primitive type. You drag, split, resize, and arrange them however makes sense for what you're doing — and Senkani remembers, per project.

No workspaces to save. No layouts to name. Close the app, reopen it, everything is exactly where you left it.

A proper terminal. SwiftTerm native, Apple Silicon GPU-accelerated, sub-3ms render latency (measured by senkani bench). True color, ligatures, Powerline. Claude Code tool calls are intercepted and compressed automatically — you don't change how you work.

Context awareness

Your context window is a resource.

Every session, Senkani tracks what it compressed, how many tokens it saved, and what it cost. The analytics pane shows you the breakdown in real time: filter savings, indexer savings, secrets redacted. Not estimated — measured from the actual byte deltas.

The session database stores the full history in SQLite with FTS5 full-text search. senkani stats gives you lifetime numbers. The analytics pane charts them session by session.

For everyone

The best developer
tool for people
who (aren't developers)*

*and for those who are.

The dirty secret of "developer tools" is that most of them punish anyone who dares not already be an expert. Senkani was built for the inverse: a friend, a designer, an analyst, or your past self from five years ago can open it and immediately do something real. A terminal. A note. A webpage. An agent.

And then — and this is the important part — it does not stop being useful as you get better. The compression layer learns what you do. The symbol index covers every language you add. The agents get access to more of your workflow. The pane system rewards layouts you couldn't have imagined on day one. It is a tool that grows with you instead of being replaced by one.

This is the thing Jobs understood that most of Silicon Valley still hasn't: approachable and powerful are not opposites. They are the same axis, and most tools live at neither end of it.
