Concept · Explanation

What is senkani?

Senkani is a native macOS binary that sits between your AI coding agent (Claude Code, Cursor, Copilot, any MCP-compatible tool) and your filesystem. Every tool call — read, grep, shell, build, search — is intercepted by a hook relay, answered by a local symbol index or filter pipeline where possible, and forwarded with only the output that matters. The agent uses less context; the agent costs less; the agent is faster.

The one-sentence version

Senkani is a trust boundary. Your agent on one side, your filesystem on the other; the boundary compresses, redacts, caches, and learns.

The category

Senkani is a new category of developer tool. It is not a new coding agent, not an IDE plugin, not a cloud service, not a framework your code imports. It is a local compression-and-context layer that makes the agent you already use cheaper, faster, and safer — by intercepting the tool calls the agent makes on your behalf.

Who it's for

  • Developers running coding agents on real codebases. If your Claude Code session is eating 40k+ tokens on a medium refactor, the compression layer pays for itself on the first prompt.
  • Operators who care about trust boundaries. Secret redaction, SSRF hardening, prompt-injection scanning, and socket-auth handshakes all ship on by default.
  • People tired of configuring LLM stacks. senkani init, one command, idempotent, zero config files required.

How the pieces connect

Senkani has three layers, and all three ship in the same binary:

  1. MCP tools — 19 tools the agent can call directly. Compressed reads, filtered shells, symbol lookups, sandboxed parse, local embed/vision.
  2. Hook relay — intercepts the agent's built-in Read/Bash/Grep and routes to senkani equivalents or denies with cached results.
  3. Smart denials — re-read suppression, command replay, trivial routing, search upgrade. The filter decides a tool call doesn't need to run at all.

What it replaces (or augments)

Senkani doesn't replace your agent — it replaces the noise your agent would otherwise send to the model. Concretely:

  • Read → senkani_read (compressed, cached, redacted) or senkani_fetch (symbol-only).
  • Grep → senkani_search (local tree-sitter index; 99% fewer tokens than text grep).
  • Bash → senkani_exec (24+ command-specific filter rules; mutating commands pass through unchanged).
  • Per-repo "what's in this codebase?" briefs → senkani_bundle (budget-bounded Markdown or JSON).
  • Cloud embed / vision API calls → senkani_embed + senkani_vision on Apple Silicon.

What "compression layer" means concretely

Three separate compression effects, all measurable via senkani bench or the live Analytics pane:

  1. Input compression. Tool outputs are shorter before they reach the model. git clone output: 312 lines → 4. npm install: 428 → 2. File reads return outlines (symbol + line numbers) by default; full content only on full: true.
  2. Redundancy elimination. The Layer-3 hook denial catches re-reads of unchanged files, deterministic re-runs of pure commands, and trivial calls like pwd/ls — and answers from cache or the deny reason itself.
  3. Output compression (terse). When enabled per pane, a system-prompt injection asks the model to strip filler phrases; a post-filter strips them on the way in too. Works synergistically with the input layers.
⚐
Savings multipliers vary. The 80.37× figure on the landing page is the synthetic fixture ceiling. Live sessions typically land in the 3–15× range depending on workflow. The Savings Test pane shows both side by side, and the live number is the honest one.

What it's explicitly not

  • Not a new model. You keep using Claude, Cursor, Copilot, Codex — senkani doesn't replace them.
  • Not a framework. Your application code doesn't import senkani. It runs beside your agent, not inside your app.
  • Not a SaaS. There's no senkani.com API. Everything runs on your machine.
  • Not a router. The per-pane model presets (options) don't wrap model calls — they hint which model each pane starts with.
  • Not a plugin host. Senkani itself is the thing. It auto-discovers skills inside your project, but it doesn't have a plugin marketplace.
