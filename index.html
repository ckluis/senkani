Positioning

What it is. What it isn't.

Senkani is a trust boundary between your agent and your filesystem. It's not a new agent; it's not a framework; it's not a cloud service. It's a local relay that makes the agent you already use cheaper, faster, and safer.

What you get.

Every layer lives in the same Swift binary. No extra processes to manage. Toggle per-pane; override per-env var; reset with one command.

01 · COMPRESSION

Perception shrinks before the LLM sees it.

24 shell-output filters, symbol-level reads, smart hook denials for redundant re-reads. The agent calls tools; Senkani's three layers return the same answer in 5–100× less text.

Read the concept → 02 · WORKSPACE

17 panes. One canvas. Your layout.

A horizontal canvas of typed panes — Terminal, Code Editor, Browser, Hook Timeline, Analytics, Knowledge Base, Sprint Review, and more. Each pane persists per-project; ⌘K opens any of them.

Tour the panes → 03 · COMPOUND LEARNING

It gets better the more you use it.

Filter rules, context docs, instruction patches, and workflow playbooks mined from your own sessions. Proposals flow .recurring → .staged → .applied; nothing auto-applies instructions without your OK.
